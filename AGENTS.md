# AGENTS.md — GB Pricing

## Проект

Сайт для сотрудников с розничными ценами на каталог RM Group.

- Репозиторий: https://github.com/Noo001/gb-pricing
- Папка: `C:\repos\gb-pricing`
- Стек: PHP 8.2+ (без фреймворков), PostgreSQL 16, чистый HTML/CSS
- Хостинг: Бегет, поддомен `pricing.salevrn.ru`
- База данных: облачная PostgreSQL 16.4 в панели Бегета

## Что уже сделано

- Авторизация: роли `admin` и `seller`.
- Админка: настройки API, управление пользователями, наценки, лог синхронизаций, ручной запуск синхронизации.
- Приём каталога по API RM Group (`pricelist.get`) через `cron/sync.php`.
- Приоритет наценок: товар → категория → глобальная → наценка по умолчанию из настроек.
- Округление цены вверх до заданного шага (по умолчанию 100 ₽), с сохранением копеек на промежуточном этапе.
- Удаление товаров, которых больше нет в выгрузке API, при успешной синхронизации.
- Страница продавца с поиском и фильтрами.
- Локальное тестовое окружение: `docker-compose.test.yml` + мок API.
- Базовая защита: `session_regenerate_id` при входе/выходе, HttpOnly/SameSite для cookie, CSRF на формах, защита от самоудаления админа.

   - настроить cron на `php /home/m/mastak97/pricing.salevrn.ru/cron/sync.php` каждые 5–10 минут.
2. Проверить синхронизацию и цены в рабочее время API RM Group (будни 11:00–18:00 МСК).

## Последние правки (30.06.2026)

- DB-слой полностью переписан с PDO на нативные `pg_*` функции (`src/db.php`).
  - На shared-хостинге Бегета PDO не мог резолвить хост облачной БД, а `pg_connect` работает стабильно.
  - Добавлена обёртка `PgPdo`/`PgStatement`, совместимая по интерфейсу с используемыми методами PDO.
  - Все SQL-запросы с плейсхолдерами `?` автоматически конвертируются в `$1, $2, ...`.
- Обновлён `src/config.php`: новый хост БД `inachagufiek.beget.app`, `sslmode=disable`.
- Сброшен и протестирован пароль администратора; `install.php` удалён с сервера.

- `public/admin/index.php` и `public/admin/sync_log.php`: ошибки синхронизации выводятся без HTML-тегов и JSON, с переносами строк.

## Структура

```
public/          # web root
src/             # PHP-логика, конфиг, авторизация
cron/            # скрипт синхронизации
sql/             # схема БД
docker/test/     # тестовое окружение
docker-compose.test.yml
```

## Запуск локального теста

```bash
cd /mnt/c/repos/gb-pricing
docker compose -f docker-compose.test.yml up -d
```

Сайт доступен на `http://localhost:8081` внутри WSL.

После подъёма контейнеров нужно один раз применить схему:

```bash
docker exec -i gb-pricing-db-1 psql -U gb_pricing -d gb_pricing_test < sql/schema.sql
```

## Важно

- `src/config.php` в `.gitignore`. На сервере файл нужно создать вручную.
- Корневой `.htaccess.deprecated` не нужен при деплое с `document root = public/`; можно удалить.
- Перед продакшеном обязательно удалить `public/install.php`, если он остался после установки.
